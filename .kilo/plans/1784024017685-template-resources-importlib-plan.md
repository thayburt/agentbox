# Move generated defaults to package resources

## Goal and constraints

- Move every user-facing generated-file template out of Python strings: the root `agentbox.toml`, Codex and Kilo TOML sections, both Containerfiles, and Kilo's injected `kilo.jsonc`.
- Load templates through `importlib.resources`; never derive filesystem paths from `__file__`, so installed and zip-imported packages work.
- Preserve generated bytes for identical settings, including blank lines, continuation backslashes, and the final LF. Existing managed-image digests must not change.
- Preserve create-once behavior: existing `agentbox.toml`, Containerfiles, and injected configs remain user-owned and are never refreshed.

## Implementation plan

1. **Add package-owned native template files** under `src/agentbox/templates/`, mirroring their ownership:
   - `agentbox.toml`
   - `codex/agentbox-section.toml`
   - `codex/Containerfile`
   - `kilo/agentbox-section.toml`
   - `kilo/Containerfile`
   - `kilo/kilo.jsonc`
   Extract the current strings exactly. Use explicit low-collision tokens such as `@@DRIVER_SECTIONS@@`, `@@BASE_IMAGE@@`, `@@IMAGE_NAME@@`, `@@WORKSPACE_FOLDER@@`, and `@@CODEX_HOME@@` only where runtime substitution is currently required. Keep `kilo.jsonc` token-free.

2. **Add a small resource renderer** (for example `src/agentbox/template.py`) that:
   - Anchors `importlib.resources.files()` at the `agentbox` package and reads descendants of `templates/` as UTF-8 text.
   - Accepts a relative resource name plus a mapping of token names to string values.
   - Discovers tokens in the original template, rejects missing or unexpected replacements, and substitutes in one regex pass so token-like text inside replacement values is not reprocessed.
   - Returns text without stripping, dedenting, or newline normalization.
   Let missing packaged resources fail clearly at the point of use; packaging validation below prevents release artifacts with absent files.

3. **Switch configuration rendering to resources** in `src/agentbox/config.py`, `src/agentbox/drivers/codex.py`, and `src/agentbox/drivers/kilo.py`:
   - Render each driver's TOML section from its resource using the same defaults and environment lookup as today (including host `CODEX_HOME`).
   - Keep the driver registry ordering and existing `rstrip()`/join behavior when composing sections, then inject the result into the root TOML template.
   - Render each Containerfile with the validated driver's configured `base_image`.
   - Load Kilo's JSONC resource for `InitFileSpec.contents`; remove `AGENTBOX_CONFIG_CONTENTS` but retain `AGENTBOX_CONFIG_RELATIVE_PATH` because runtime mounts and diagnostics use it.
   - Keep driver protocols, `InitFileSpec`, Podman hashing/build logic, snapshot logic, and CLI write-once flow unchanged.

4. **Make wheel inclusion explicit** in `pyproject.toml` with Hatch's wheel package selection for `src/agentbox`, ensuring the resource subtree is shipped with the package. Confirm the sdist also contains the templates; add only the minimal Hatch sdist inclusion setting if artifact inspection shows the defaults do not already include them.

5. **Add focused regression coverage** (preferably `tests/test_templates.py`, plus small updates to existing config/driver tests):
   - Render both Containerfiles with a non-default base image; assert the first line changes, no tokens remain, continuation backslashes are literal, and the exact final newline remains.
   - Render driver sections with non-default values, including `CODEX_HOME`, and verify root `default_toml()` remains valid via `tomllib` with the expected section ordering and values.
   - Assert Kilo `init_files()` returns the exact JSONC resource text and its existing destination.
   - Keep existing CLI/Podman tests proving files are created independently, not overwritten, hashed, and snapshotted.
   - Add a resource-reader failure/mismatched-token test so malformed template/consumer pairs fail deterministically.

## Validation

1. Run `uv run python -m unittest discover -s tests -v`.
2. Before/after the extraction, compare the rendered defaults for both drivers and `default_toml()` byte-for-byte under a controlled environment; compare their SHA-256 digests as an additional guard.
3. Run `uv build`, inspect both sdist and wheel for all six template files, then install the wheel into a clean temporary environment and call `default_toml()`, both `default_containerfile()` methods, and Kilo `init_files()` to prove resource access does not depend on the source checkout.

## Risks and rollout

- A dropped LF or changed backslash changes newly generated files and managed-image tags; byte comparison is mandatory.
- Missing wheel data would become a runtime resource error; explicit package selection plus clean-wheel smoke testing covers this.
- No data migration is needed. Existing repository-local generated files remain untouched; only future file creation reads the packaged resources.
